# SEO & Discoverability Audit — agamarora.com
**Date:** 2026-05-01  
**Auditor:** Claude Sonnet 4.6 (read-only static analysis)  
**Scope:** All public routes — /, /lab, /lab/second-brain/, /lab/ai-resume/, /lab/voice-ai-production/, /lab/product-shape/, /resume, /enter, /wiki/graph/, 12 wiki theme pages, 19 beliefs sub-pages, /wiki/themes/, /wiki/beliefs/, /404

---

## Executive Summary

1. **39 pages share `/assets/og/lab.png` as the og:image** — every wiki theme page, every belief sub-page, /wiki/themes/, /wiki/graph/, and /lab all unfurl with the same generic image. Zero differentiation in social previews for the richest content on the site.
2. **No search engine ownership verification** — Google, Bing, Yandex verification tags are absent from every page. No IndexNow integration. Zero GSC/Bing WMT data flowing.
3. **No BreadcrumbList JSON-LD on nested routes** — wiki pages have visible breadcrumbs in HTML but zero structured BreadcrumbList schema. Google cannot render breadcrumbs in SERPs.
4. **`/resume` and `/enter` are missing og:image entirely** — both pages render blank unfurls on LinkedIn, Twitter, and Slack. `/resume` also has no `twitter:card`.
5. **Person schema `sameAs` is missing Medium and Calendly** — two of the owner's authoritative profile URLs are absent from the Knowledge Panel trigger array.

## Dimension 2: Structured Data (JSON-LD)

### Person Schema (landing page `index.html:38–108`)

**What's present:** `@type: Person` with name, givenName, familyName, url, image, description, jobTitle, worksFor, alumniOf, knowsAbout, knowsLanguage, hasOccupation, sameAs. WebSite schema present.

**Finding [P1]** `index.html:91–95` — `sameAs` array contains only LinkedIn, GitHub, YouTube. Missing: **Medium** (`https://medium.com/@agam.arora11`) and **Calendly** (`https://calendly.com/agamarora/chat`). For a Knowledge Panel, Google cross-references sameAs against known entity pages. Medium is a publishing platform that strengthens topical authority signals. Fix: add both to the sameAs array.

**Finding [P1]** `index.html:49` — Person schema `image` points to `https://agamarora.com/assets/preview.png` — this is a 1200×630 banner/og-image format, not a headshot. Google's Person schema `image` is supposed to be a portrait/headshot for Knowledge Panel. Fix: add a dedicated `https://agamarora.com/assets/agam-headshot.jpg` (if one exists or can be created) as the schema image; keep the preview.png as og:image.

**Finding [P1]** `index.html:98–105` — WebSite schema is missing `potentialAction` → `SearchAction`. This enables Google Sitelinks Searchbox in SERPs (when query volume threshold is met). Fix: add:
```json
"potentialAction": {
  "@type": "SearchAction",
  "target": { "@type": "EntryPoint", "urlTemplate": "https://agamarora.com/enter?q={search_term_string}" },
  "query-input": "required name=search_term_string"
}
```

### Article Schema (wiki theme pages)

**Finding [P1]** All wiki theme pages and belief sub-pages (18+ files) — Article schema present but missing `datePublished` and `dateModified`. These are required by Google's Article rich result spec. Without them, pages are ineligible for article rich results. Fix: add static dates to the JSON-LD in the build script (`scripts/build-wiki.mjs`), e.g. `"datePublished": "2026-04-27"`.

**Finding [P1]** All wiki Article schemas — missing `image` field. Article schema `image` is required for Google News and Discover eligibility. Fix: set to the wiki og:image URL for each page (or a default wiki banner).

**Finding [P2]** All wiki Article schemas — `headline` field contains double-space characters (` - ` → `  -  `) matching the title tag pattern. Technically valid but inconsistent and the extra spaces will appear verbatim in structured data validators. Fix: normalize to single-dash ` — ` (em-dash) or ` - ` (single space each side).

### BreadcrumbList

**Finding [P0]** Zero pages have BreadcrumbList JSON-LD anywhere on the site. Wiki theme pages at `/wiki/<slug>/` have visible breadcrumb HTML (`wiki / {page-name}`) but no schema counterpart. Belief pages at `/wiki/beliefs/<slug>/` have two-level nesting. Lab sub-pages are at `/lab/<slug>/`. Google cannot render breadcrumb trails in SERPs without the schema. Fix: add BreadcrumbList JSON-LD to: all wiki theme pages, all beliefs pages, all lab sub-pages. The build script already generates wiki pages; add the BreadcrumbList block there.

### ProfilePage Schema (`/resume`)

**Finding [P1]** `resume/index.html:15–30` — ProfilePage schema present with correct mainEntity reference. However, missing `about` property pointing to `@id: "https://agamarora.com/#person"`. Also: `mainEntity` uses inline Person object instead of `@id` reference — this severs the graph connection to the root Person node. Fix: change `mainEntity` to `{ "@type": "Person", "@id": "https://agamarora.com/#person" }` (drop the `name` and `url` inline, let the @id resolve).

### FAQPage

**Finding [P2]** 19 belief sub-pages each have FAQPage schema with exactly one Q&A about the belief. Schema is technically valid but single-question FAQPage won't trigger FAQ rich results (Google typically requires 2+ distinct Q&As). Fix: add a second Q&A to each belief page, e.g. "How did this belief form?" with answer text from the origin section.

## Dimension 6: Search Engine Ownership

**Finding [P0]** No Google Search Console verification tag found on any page. Without GSC verification, there is no access to: crawl errors, index coverage, Core Web Vitals field data, manual actions, search query performance, or sitemap submission status. Fix: verify via HTML tag (`<meta name="google-site-verification" content="...">`) in `index.html` head, or via DNS TXT record (preferred — no code change needed).

**Finding [P0]** No Bing Webmaster Tools verification (`msvalidate.01`) on any page. Bing (and Copilot/ChatGPT Browse) use Bing's index. Without WMT verification, no sitemap submission, no crawl data, no ChatGPT-Browse citation analytics. Fix: add `<meta name="msvalidate.01" content="...">` to `index.html`.

**Finding [P2]** No Yandex verification (`yandex-verification`). Lower priority for an English personal site targeting US/global audience, but Yandex is a significant LLM training data source. Fix: add if international reach is desired.

**Finding [P2]** No IndexNow API key. IndexNow allows instant index notification to Bing, Yandex, Seznam on content change. Since Netlify auto-deploys on push, a build plugin could POST to IndexNow on each deploy. Fix: add an IndexNow key file and a Netlify build plugin to notify on deploy.
