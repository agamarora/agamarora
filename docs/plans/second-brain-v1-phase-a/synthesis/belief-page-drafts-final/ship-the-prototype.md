# Ship the prototype

## Statement

Ideas are cheap. Ship the prototype.

Don't pitch the idea - ship a working version of it. The prototype is the position. No longer "just an idea!" is the inversion marker. The belief dates to 2020-05-19, when the phrase appears in a post, directly naming the prior identity and walking away from it.

The loop closes when it's posted. Build, post, teach, learn, build again. Posting is the step that converts personal projects into reusable artifacts and feedback. Without the post, the build stays private. With it, the build compounds.

## Origin

**Pre-2020:** Gaming studio work, operational rebuilds. Shipping was already the behavior. The framing was not yet "prototype-first" explicitly - it was operational and business-shaped.

**2020-04-26:** Lockdown removed the external dev capacity for a project idea. Response: pull up a Flutter/Dart course and acquire the capability personally. "Tough times call for action more than anything. Wasn't able to invest into an app development team for an idea of mine. Pulled up this brilliant app development course on Introduction to Flutter development using Dart by Dr. Angela Yu and got it done with. Off to creating."

**2020-05-19 (canonical post - the inversion):** "Sharing that I am ready to take up prototyping projects for cross platform app development. This gives me the superpower to test and implement ideas on my own. Now creating a prototype as a proof-of-concept is much easier. With dart/flutter I am able to create 'quick and dirty', yet 'powerful and clean' looking mobile app prototypes (fully working applications or MVP if you may) to go along a business pitch. No longer 'just an idea!' :)"

Two things make this the supersession date: it is explicit ("No longer 'just an idea!'" names the prior identity and closes it) and it is substrate-backed (capability acquired before the declaration - Agam did not just say "ideas need prototypes," he built the prototyping capability first).

## Refinement arc

The belief has not been superseded. It has been refined upstream.

**2020-09-17 era:** Reid Hoffman, reposted. "If you are not embarrassed by the first version of your product, you've launched too late." Borrowed authority for the same claim: ship even when embarrassed.

**2023-04-16:** agamarora.com v1, Python plus Streamlit, GitHub repo in the post. "How much time does it take to build your own personal website? It took me 12 years!" Confessional shipping voice. The link is in the post. Late arrival is still arrival.

**2024 (AI accelerates the cadence):** "Here are a few AI projects you can do over the weekend: 1/ setup ollama and run local LLM models, 2/ try agent frameworks like autogen and crewAI, 3/ create GPTs for different use-cases." Not "here are ideas to think about" - "here are projects to ship over a weekend." And: "AI building AI is now :)" The substrate reduces the barrier to each ship.

**2026-04-09 (upstream tightening via spec-over-sprint):** Spec over Sprint, Taste over Execution, Context over Prompt. `belief.spec-over-sprint` enters. The key clarification: it refines, does not supersede. 2020 position: "ship ANY prototype, beats no prototype." 2026 position: "ship a well-spec'd prototype." The verb stays constant. The adjective tightens.

**2026-04-23 (latest instance):** "Plain markdown. Git. Open source. MIT. Free forever. Link in the comments." Eight years from the 2018 gaming-retrospection seed to a shipped public artifact.

**Tinker-publicly absorbed (per taste-pass A3):** Tinker-publicly is not a separate belief - it is the social register of this one. Every shipped prototype in the corpus was also publicly posted (evidence sets overlap 90%+). The merge clarifies: posting is not distribution. It is the step that makes the build reusable. The loop is one thing.
