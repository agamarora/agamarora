# Learn concepts, not tools. The durable IC investment in an era of tool churn.

Frameworks endure. Tools rotate. STP (Kotler, via Wendell Smith, 1956) is still live in AI-PM practice today. The concept has not changed. The tools running segmentation have changed dozens of times. Investing in the tool layer is renting. Investing in the concept layer is owning. AI makes the rotation speed faster, not slower. The half-life of tool-specific knowledge is compressing. The concept underneath stays constant.

## The three-type taxonomy

Three types of practitioners: the driver uses the tool; the mechanic repairs it; the engineer knows it inside out and can create similar tools from first principles. AI commoditizes drivers. "Being technical is not about knowing a technology but using the technology." Concept-application over tool-knowledge. The engineer survives the rotation because they can evaluate, adopt, and deprecate tools against the underlying concept without losing their footing. The practitioner who architected their skill stack around concepts has shipped value through four tool generations while the driver constrained to one toolchain relearned from scratch each time.

The application to AI tooling is direct. Do not learn a specific orchestration framework; learn how agents reason, how context windows shape output, how retrieval changes answer quality. The framework changes with each model generation. The reasoning structure does not. The operator who understands context management as a concept can evaluate any new RAG implementation in an afternoon. The operator who learned one specific tool has to relearn from scratch.

## Concept depth at the AI PM layer

The PM skill set includes concepts that AI cannot commoditize: JTBD (what is the underlying job the user is hiring this product for?), NFR specification (what are the non-functional requirements that block enterprise compliance?), V/V/U framing (viable, valuable, usable), RWDA (real, winnable, worth-it, aligned). These frameworks predate every current AI tool. They will outlast every current AI tool. Depth on these is the IC's durable advantage.

The learning investment that pays: when AI commoditizes a tool layer, identify the concept the tool was running. ChatGPT commoditized surface-level research. It did not commoditize information architecture, synthesis judgment, or knowing which framework applies to which problem class. Those concepts are the durable layer. Double down there.

## Concept fluency as the hiring signal

A PM candidate who reaches for named frameworks in technical discussions is demonstrating that concepts are internalized, not just recalled. STP cited to the professor who wrote it on a whiteboard in 1956. Greene's Mastery cited on deliberate practice. Frameworks deployed as the answer architecture, not as a vocabulary flex. That signal is visible to senior hiring managers and invisible on a resume that lists tool names.

The diagnostic: audit the learning diet for concept-to-tool ratio. A course that is 80% tool-specific (shortcuts, UI flows, vendor features) has a short shelf life. One that is 80% concept-specific (mental models, decision frameworks, system architectures) compounds across the career. The ratio is the test. Most candidates get it wrong.

## Bottom line

Learn-concepts-not-tools is an IC career economics principle. Tool fluency earns the first interview. Concept depth earns the promotion and the next role after that. At the AI-era operator level, tools rotate faster than job cycles. The PM who invested in JTBD, NFR specification, and agent architecture as concepts can evaluate any new tool that implements them in an afternoon. The PM who invested in tool fluency has to relearn with each generation. The concept layer compounds. The tool layer depreciates.
